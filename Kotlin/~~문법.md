# Kotlin 문법

## 변수

#### 기본형
```kotlin
val a: Int = 1
```
#### 타입 생략 가능
```kotlin
val b = 1
```
#### 선언 & 초기화 분리 가능
```kotlin
val c: Int
c = 3 // 지연 할당시 타입 명시 필요 
```

### val
- value
- 값 초기화 이후 재할당 불가
- read-only
- 객체 자체의 불변성을 보장하지는 않음
- Java의 Final과 유사
```kotlin
val s = "변경 불가"
//s = "변경 가능?" // 컴파일 불가
```

### var
- variable
- 값 재할당 가능
- 타입 변경 불가
```kotlin
var s = "변경 가능"
s = "변경 가능 22"
```

### 탑 레벨 선언
- top-level declaration
```kotlin

    println(i)//2
}
```

## 함수
#### 기본형
```kotlin
fun sum(a: Int, b: Int): Int {
    return a + b
}
```
#### 표현식
```kotlin
fun sum(a: Int, b: Int): Int = a + b
fun sum2(a: Int, b: Int) = a+b// 타입 추론이 가능한경우 반환타입 생략 가능
```
#### Unit
- 반환타입이 없는 함수의 기본형
```kotlin
fun a (a:Int) {
    println(a)
}
```
- 의미 있는 반환값이 없는 함수의 반환 타입
- 반환 타입이 `Unit`이면 일반적으로 생략 가능
```kotlin
fun b (a: Int): Unit {
    println(a)
}
```

#### 디폴트 파라미터 
- Default Parameter
- 인자를 받지 않은 경우 디폴트 파라미터가 존재하는 경우 디폴트 값으로 작동함
```kotlin
fun main() {
    hi()//안녕!

}

fun hi(message : String = "안녕!") {
    println(message)
}
```

## 반복문
### For
#### ForI
```kotlin
for (i in 0 .. 3){
    print(i)
}//0123
```
```kotlin
for (i in 0 until 3){
    print(i)
}//012
```
```kotlin
for (i in 0 .. 6 step 2){
    print(i)
}//0246
```
- step은 양수만 사용(음수인 경우 IllegalArgumentException 발생)
- 감소가 필요한 경우 downTo 사용
```kotlin
for (i in 3 downTo 0){
    print(i)
}//3210
```
```kotlin
for (i in 6 downTo 0 step 2){
    print(i)
}//6420
```

- Java와 비슷하게 사용
```kotlin

    if(a>5) println("5보다 큼")
    else println("5와 같거나 작음")
}
```
#### 표현식
- If 문법은 표현식임
```kotlin
val a = 5
val 판별: String =
    if (a > 5) {
        "a는 5보다 큼"
    } else if (a < 5) {
        "a는 5보다 작음"
    } else {
        "5와 같음"
    }
```
- 삼항 연산자가 존재하지 않음

### When
- Switch 문법과 비슷
```kotlin
val day = 1
val result = when (day) {
    1 -> "월요일"
    2 -> "화요일"
    3 -> "수요일"
    4 -> "목요일"
    5 -> "금요일"
    else -> "주말"
}
```
#### 모든 경우를 처리하면 else 생략 가능
```kotlin
fun main() {
    val day = true
    val result = when (day) {
        true -> 1
        false -> 2
    }

    val color = when (getColor()) {
        Color.RED -> 1
        Color.BLUE -> 2
    }

}

enum class Color {
    RED, BLUE

}

fun getColor() = Color.RED
```
#### 여러개의 조건을 한줄로 지정 가능
```kotlin
val day = 1
when (day) {
    1, 0 -> println("1 이나 0")
    else -> println("1 이나 0이 아님")
}
```

## Null 처리
- 언어적 차원에서 Null의 발생 가능성을 제거하려 노력
```kotlin
val a: String = null//컴파일 에러
var b: String = "text"
b = null//컴파일 에러
```

#### 안전 연산자(?)
- 안전 연산자를 통해 Null 접근이 가능

 ```kotlin
val a: String? = null
// a.length // 컴파일 오류
a?.length // null
```
#### 엘비스 연산자
- 왼쪽 값이 null 인 경우 오른쪽 값을 사용
```kotlin
val c = a?.length ?: 0
```
#### 단언 연산자
- nullable 값을 null이 아닌 타입으로 강제 변환
- 실제 값이 `null`이면 실행 중 `NullPointerException` 발생
```kotlin
val c: String? = null
val d = c!!.length
```

## Exception
- CheckedException을 강제하지 않음
```kotlin
fun main(){
    Thread.sleep(1)
}
```

#### try catch
```kotlin
fun main(){
    try{
    Thread.sleep(1)
    } catch (e : Exception){
        println("예외 발생")
    }
}
```
#### finally
```kotlin
fun main(){
    try{
    Thread.sleep(1)
    } catch (e : Exception){
        println("예외 발생")
    } finally {
        println("Finally")
    }
}
```
#### try를 표현식으로 사용
```kotlin
fun main(){
    val a = try{ 
        "abc".toInt()
    } catch (e : Exception){
        0
    }
}
```

### 상속
- Kotlin의 모든 non-null 클래스는 기본적으로 `Any`를 상속
- `Any`는 `equals()`, `hashCode()`, `toString()`을 제공
- 기본적으로 상속을 막아놓아 상속이 필요한 경우 open 을 사용하여 상속 가능
```kotlin
open class Dog {
    open var age: Int =0
    open fun bark(){
        println("멍멍")
    }
}

class Bulldog : Dog(){
}
```

#### override
```kotlin
class Bulldog(override var age: Int) : Dog(){ // 생성자에도 override 가능
    override fun bark() {
        println("왕왕")
    }
}
```
- override 한 속성은 open으로 변경되어 하위 클래스에서 재정의하지 못하도록 막아야 할 경우 override 앞에 final 선언
```kotlin
open class Bulldog(override var age: Int) : Dog(){
    final override fun bark() {
        println("왕왕")
    }
}
```
#### super
```kotlin
open class Bulldog(override var age: Int) : Dog(){
    override fun bark() {
        super.bark()
    }
}
```
#### abstract

```kotlin
abstract class Animal {
    abstract var age: Int
    abstract fun introduce(name: String)
}
class Human(override var age: Int) : Animal() {
    override fun introduce(name: String) {
        println("제 이름은 $name 입니다")
    }
}
fun main(){
    val animal:Animal = Human(age = 20)
    animal.introduce("홍길동")
}
```
### Interface
```kotlin
interface Animal// 인터페이스 구현은 ()를 사용하지 않음

interface Beverage {
    var price: Int
    fun make(quantity:Int)
    fun doSomething()
}
```
#### 인터페이스 구현
```kotlin
class Juice(override var price:Int) : Beverage{
    
    override fun make(quantity: Int) {
        println("주스 $quantity개를 만듭니다.")
    }
    
    override fun doSomething(){
        println("doSomething")
    }
}
```

## Data
```kotlin
data class Person(val name: String, val age: Int)
```
-  equals, hashCode, toString 등의 메소드를 자동으로 만들어줌

### component n
```kotlin
fun main(){
    val p1 = Person("길동",20)
    print("${p1.component1()},${p1.component2()}")//길동 20
}
```
#### 구조분해 할당에 내부적으로 사용됨
```kotlin
fun main() {
    val (name,age) = Person("길동",20)
    println("$name,$age")//길동 20
}
```
